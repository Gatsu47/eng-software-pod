2  * @file ATA6870__INTERRUPTS.C
3  * @brief Interrupt handlers for the ATA6870 device driver
4  *
5  * @note
6  * In NORM mode (normal mode), the reasons for an interrupt request are:
7  * 1. The availability of measured data (data ready)
8  * When a voltage measurement is completed, the dataRdy flag is set in the status register.
9  * The ATA6870N cannot decode any new incoming operation until the dataRdy flag is released.
10  * 2. The low frequency timer (LFT) elapses (wakeup)
11  * The wakeup flag is set in the status register when the LFT elapses. The LFT is controlled via the SPI interface.
12  * 3. A transmission error is flagged during the last SPI transaction (the commError bit is set in the status register).
13  * 4. If an undervoltage condition occurs. The undervoltage function is controllable via SPI interface.
14  *
15  * A mask bit in the irqMask register corresponds to each interrupt source. The MCU must read the ATA6870N status register
16  * before the interrupt is cleared. With each SPI access a 16-bit IRQ state is sent via MISO to the MCU with the interrupt state
17  * of all stacked ATA6870N (see Section 7.6.4.1 â€œSPI Transaction Fieldsâ€� on page 26).
18  *
19  * In PDmode (power down), if the digital control part and MCU are not supplied, neither SPI command nor interrupt are
20  * transmitted over the interface.
