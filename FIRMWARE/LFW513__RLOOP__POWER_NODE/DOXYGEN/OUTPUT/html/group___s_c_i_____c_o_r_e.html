void u8RM4_SCI__RxByteArray ( RM4_SCI__CHANNEL_T  eChannel,
Luint32  u32Length,
Luint8 *  pu8Data 
)

Rx a byte array Receive a block of 'length' bytes long and place it into the data buffer pointed to by 'data'.

If interrupts have been enabled the data is received using interrupt u32Mode, otherwise polling u32Mode is used. In interrupt u32Mode receive is setup and the routine returns imediatly, u8RM4_SCI__RxByteArray must not be called again until the transfer is complete, when the vRM4_SCI_INT__Notification callback will be called. In polling u32Mode, u8RM4_SCI__RxByteArray will not return until the transfer is complete.

Parameters
[out]pu8DataPointer to rx buffer
[in]u32LengthLength required
[in]eChannelThe SCI Channel

void vRM4_SCI__TxByteArray ( RM4_SCI__CHANNEL_T  eChannel,
Luint32  u32Length,
Luint8 *  pu8Data 
)

Send Array Send a block of data pointed to by 'data' and 'length' bytes long.

If interrupts have been enabled the data is sent using interrupt mode, otherwise polling mode is used. In interrupt mode transmition of the first byte is started and the routine returns imediatly, vRM4_SCI__TxByteArray must not be called again until the transfer is complete, when the vRM4_SCI_INT__Notification callback will be called. In polling mode, vRM4_SCI__TxByteArray will not return until the transfer is complete.

Note
if data word is less than 8 bits, then the data must be left aligned in the data byte.

[in]pu8DataPointer to byte array
